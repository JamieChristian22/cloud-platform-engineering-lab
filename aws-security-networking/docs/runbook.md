# Operations Runbook

## Daily Checks

- Review health dashboard.
- Review error rates and latency.
- Check failed authentication logs.
- Confirm no new critical security findings.
- Confirm backup status.
- Review current cloud spend.

## Weekly Checks

- Review IAM access.
- Validate alert rules.
- Review patching requirements.
- Confirm Terraform state consistency.
- Review incidents and lessons learned.

## Emergency Actions

1. Identify affected service.
2. Determine severity.
3. Preserve logs.
4. Isolate affected workload.
5. Notify stakeholders.
6. Execute rollback or recovery.
7. Validate service health.
8. Complete postmortem.
