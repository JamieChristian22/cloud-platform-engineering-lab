# AWS Multi-Account Governance Platform

## Project Overview
This project demonstrates an enterprise AWS governance platform using AWS Organizations, Organizational Units, Service Control Policies, identity governance, security guardrails, and operational documentation.

## Business Problem
Organizations need centralized governance across cloud teams, accounts, and environments. Without governance, AWS environments can develop inconsistent security, weak permissions, poor cost controls, and limited audit readiness.

## Solution
Implemented a governance model using:
- AWS Organizations
- Organization Root
- Organizational Units
- Service Control Policies
- IAM governance model
- Security guardrails
- Operational runbooks
- Evidence documentation

## OU Design
| OU | Purpose |
|---|---|
| Infrastructure | Shared networking and platform services |
| Security | Logging, auditing, compliance, monitoring |
| Workloads | Application and business workloads |
| Sandbox | Testing, labs, experiments |
